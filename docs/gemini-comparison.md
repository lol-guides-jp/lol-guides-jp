# Claude vs Gemini 使い分けメモ

> 最終更新: 2026-04-12
> 背景: Gemini API 有料化（$30/月）を検討するにあたり、両者の強みを整理

## 前提

- Claude: MAX5 契約（月額固定）。L3（判断・執筆・レビュー）が本業
- Gemini: API 無料枠（20 RPD）で開始。L1/L2（検索・大量生成）のオフロード先 + 独自機能

## Gemini だけが持っている武器

### NotebookLM
ドキュメント・PDF・動画・音声を放り込むと教材として質問に答えるツール。

- **Audio Overview**: 資料からポッドキャスト風の音声要約を自動生成
- 無料: 100 ノートブック / 3 音声生成/日
- 有料（$20/月 or AI Pro に含む）: 500 ノートブック / 20 音声/日

### ネイティブ画像生成（Nano Banana）
Gemini モデル自体に画像生成が組み込まれている。テキストと画像を混在させた出力が可能。

- Imagen 4: API で $0.02〜$0.06/枚
- Gemini 2.5 Flash Image / 3.1 Flash Image でモデル内蔵生成もある
- Claude には画像**生成**機能がない（入力のみ）

### ネイティブ動画・音声理解
動画ファイルをそのまま渡してシーン分析・要約ができる。Claude は画像入力のみ。

### Google Search グラウンディング
API レスポンスにリアルタイム検索結果 + 引用元 URL を付けられる。検索結果をモデルの推論に直接統合する。

### コンテキスト長
Gemini 3.1 Pro は 2M トークン。Claude は 200K（beta で 1M）。長大なコードベースの一括分析で差が出る。

### Deep Research Agent
自律的に「計画→検索→読解→統合」を回す調査エージェント。API（Interactions API）からも呼べる。

### Jules（コーディングエージェント）
VM 上で非同期にコード変更を実行。Gemini 2.5 Pro ベース。V2（Jitro）が 5/19 Google I/O で発表予定。

### Veo 3.1（動画生成）
AI Studio 内で動画生成ができる。

## API 面の比較

| 特徴 | Claude | Gemini |
|---|---|---|
| Context caching | prompt caching あり | キャッシュ済みトークン **90% OFF** |
| Batch API | なし | リアルタイムの **50% OFF** |
| コード実行サンドボックス | なし | 内蔵（API から利用可） |
| 無料枠 | なし | あり（RPD 制限付き） |

## 使い分け

| 仕事 | Claude | Gemini |
|---|---|---|
| 判断・執筆・レビュー（L3） | 本業 | — |
| 大量生成・検索統合（L1/L2） | — | 本業 |
| 画像生成 | 不可 | Nano Banana / Imagen 4 |
| 動画・音声の理解 | 不可 | ネイティブ対応 |
| 長大ドキュメント分析 | 200K まで | 2M まで |
| パッチノートの音声要約 | — | NotebookLM |
| 自律調査 | Agent（対話） | Deep Research（API） |
| 音声入力→記事ドラフト | — | Gemini Live で実運用中 |

## $30/月の判断基準

**今は無料枠（$0）で始める。** 有料に切り替えるタイミング:

1. RPD 20 がボトルネックになったとき（パイプライン完了日数を短縮したい場合）
2. content-pipeline でも検索系タスクを Gemini にオフロードしたいとき
3. 画像生成や NotebookLM を業務で定常的に使い始めたとき

設計上は無料→有料の切り替えはコード変更不要（RPD 制限が緩和されるだけ）。
